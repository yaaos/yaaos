# Glossary

> Shared vocabulary across backend and frontend. Terms appear in code, URLs, and UI.

| Term | Meaning |
|---|---|
| **Org** | Tenant boundary. UUID PK + immutable `slug`. Every non-user row is `org_id`-scoped. Soft-deleted via `archived_at`. |
| **Ticket** | yaaos unit of work. References a PR; flows `open → in_review → complete / abandoned`. |
| **PR** | VCS-side artefact mirrored into `pull_requests`. Owned by `domain/tickets` (a property of a ticket; table `pull_requests` unchanged). |
| **Skill** | A `SKILL.md` file checked into the repo, identified by a short name, resolved by convention at `.claude/skills/<name>/SKILL.md`. Pipeline skills carry a `pipeline-` prefix (e.g. `pipeline-code-review`) marking them as run-engine consumables rather than human commands; their JSON output contracts are committed at `.claude/skills/pipeline-schemas/`. A `skill` or `review` `Stage`'s `skill_name` (and a `ReviewConfig.skill_name`) names one this way; unvalidated at pipeline-edit time — a run against a repo missing the named skill fails loudly at that stage. Also runnable directly by a developer (BYO — the same file a pipeline invokes locally). |
| **Subagent** | A Claude Code sub-agent. The review skill may dispatch subagents internally; yaaos does not define or install them. |
| **Coding agent** | The external CLI the remote WorkspaceAgent runs (Claude Code). Protocol: `core/coding_agent.CodingAgentPlugin`. yaaos never calls an LLM directly, and never execs the CLI in-process. |
| **Workspace** | Provisioned execution environment owned by a remote WorkspaceAgent. Lifecycle owned by `core/workspace`; the only registered `WorkspaceProvider` is `remote_agent`. Dispatch via AgentCommand; org-level IAM ARN (`orgs.registered_iam_arn`) authorizes the agent pod. |
| **Finding** | Durable ticket-level entity (`domain/findings`, table `pipeline_findings`). Materialized the moment a review iteration reports it — including findings the fix loop resolves a minute later. Lifecycle `open → resolved / dismissed` (`dismissed` terminal — a re-sighting after dismissal is a new finding); `severity` immutable; handle `<category>-<NNN>` where `category` is the reporting skill's one-word functional classification (`sec`, `arch`, …) and `NNN` is a per-ticket global sequence; append-only `status_events` trail records who/when/how per transition. |
| **Lesson** | Repo-scoped institutional memory: `{title, body, source_pr_url}`, 1000-char body cap. Surfaces in agent prompts. Owned by `domain/lessons`. |
| **Plugin** | Vendor-specific implementation of a Protocol in `core/`. Three Protocols: `VCSPlugin` (`core/vcs`, github), `CodingAgentPlugin` (`core/coding_agent`, claude_code), `WorkspaceProvider` (`core/workspace`). Vendor SDKs only in `apps/backend/app/plugins/`. |
| **Verdict** | Terminal review state: `APPROVED / CHANGES_REQUESTED / COMMENT`. Decided by the CLI; returned in the agent's stdout JSON. |
| **Skip reason** | Why a job didn't run: `fork`, `bot_author`, `trivial_diff`, `too_large`, `secrets_detected`, `ui_cancel`, `superseded`. |
| **Audit entry** | One append-only row in `audit_log`. Kind: `<entity>.<verb_past>`. Payload: Pydantic model owned by the writing module. |
| **Actor** | Who initiated an action — `{kind: "github_user" \| "agent" \| "system" \| "user" \| "workspace" \| "sso", login?, agent_id?, user_id?, workspace_id?}`. Required on every audit entry. Defined in `core/audit_log`. |
| **Onboarding** | Dashboard pre-ready state. Two checks: GitHub App installed + Anthropic API key validated by live probe. Computed by `domain/orgs.get_onboarding_status()`. |
| **User** | Human identity. UUID PK, soft-deletable via `deactivated_at`. Owned by `core/identity`. |
| **Membership** | `(user_id, org_id)` link. Carries per-org `handle` and one `Role`. |
| **Role** | `owner ≥ admin ≥ member`. Compared via `role.covers(required)`. Minimums declared at `Depends(require(Action.X))` call sites. |
| **Session** | Opaque server-side row keyed by `sha256(raw_token)`. Cookie: `HttpOnly; SameSite=Lax; Secure`. Double-submit CSRF via per-session `csrf_token`. `sso_satisfied_for_org_id` + timestamp track 8h SSO TTL. |
| **Invitation** | Owner/Admin-issued offer for an external email. Token: `itsdangerous`-signed, 7-day TTL, single-use. Stored as `sha256(token)`; raw lives only in the invitation email. |
| **Provider** | OAuth identity provider. Plugins implement the `Provider` Protocol; return `ProviderProfile` (`external_subject`, `primary_email`, `email_verified`, `display_name`, `mfa_satisfied`). |
| **SSO** | SAML 2.0 per-org single sign-on. SP-initiated. Satisfied assertion → `sso_satisfied_for_org_id` on session. Middleware blocks org access when SSO is enabled and session isn't satisfied. |
| **Break-glass Owner** | SSO `exempt_owner`. Signs in via OAuth + TOTP when SSO is broken. Candidate must have a verified TOTP secret. Each bypass audits `break_glass_exempt_owner`. |
| **VCS plugin** | Implements `core/vcs.VCSPlugin`. Ships `github` only. At most one per org; state on `orgs.vcs_plugin_id` + `orgs.vcs_settings`. |
| **Plugin install** | A specific `(org_id, plugin_id)` adoption. Mutations audit (`vcs.installed / vcs.cleared / coding_agent.installed / coding_agent.uninstalled`). |
| **Verified GitHub username** | `users.github_username`. Denorm written by the OAuth callback on every sign-in. Re-binding is "sign in with GitHub again." Never user-typed. |
| **Session-timeout override** | Nullable `orgs.session_timeout_override` (minutes). `require()` dep rejects sessions past `last_seen_at + override`. Null falls back to `SESSION_IDLE_TIMEOUT` (12h). |
| **BYOK** | Bring-your-own-key per `(org_id, provider)`. Ships `anthropic` only. Plaintext only inbound on POST + outbound from `core/byok.get/.validate`; DB stores Fernet ciphertext. |
| **MCP** | Model Context Protocol — JSON-RPC-over-HTTP shape hosted integrations (Linear, Notion) speak to coding-agent CLIs. yaaos proxies every MCP request via `domain/mcp_proxy`. |
| **MCP review token** | Opaque bearer authenticating the CLI to the yaaos proxy (`domain/mcp_proxy.mint_token`), scoped to a caller-chosen `review_id` (`sha256` persisted, raw returned once); revocable via `revoke_token`, swept hourly. 2h absolute TTL. |
| **Integration** | yaaos record of a connected hosted provider for an org: encrypted OAuth tokens, per-tool allowlist, status. Owned by `domain/integrations`; one row per `(org_id, provider)`. |
| **Hosted MCP** | MCP server at a provider (mcp.linear.app, mcp.notion.com). yaaos proxy forwards to `ProviderConfig.mcp_url`. |
| **Org service account** | Single upstream OAuth identity an org has connected per provider. MCP dispatch always audits `upstream_account="org_service_account"` — never the triggering developer. |
| **Allowlist** | Per-`(org_id, provider)` list of write tools the proxy will forward. Read tools always allowed; write tools opt-in (empty = read-only). |
| **Broken-creds** | Integration state when `last_refresh_status == "failed"`. Surfaces in five places today (banner, email, audit, settings badge, Claude Code warning); a per-invocation review-output prefix is tracked (`mcp_proxy.record_broken_creds`/`consume_broken_creds`) but has no consumer yet. |
| **Upstream identity** | Display string the OAuth flow returned for the connected account. Stored on `mcp_credentials.upstream_identity` for UI; never used as auth principal. |
| **Intake** | Inbound surface turning an external signal into a ticket. `core/intake` ships a typed registry keyed by name; `POST /api/intake/{type}` dispatches via the registry. `github` is the only type today. |
| **Intake point** | A plugin-contributed trigger source (`core/intake.IntakePoint`: `id`, `plugin_id`, `label`, `kind` — `webhook \| schedule`) a `Trigger binding` targets. Shipped: `github:pr_opened`, `github:pr_commits`, `github:pr_comment` (webhook, `plugins/github`), `schedule` (schedule, first-party). Listed by `GET /api/intake/points`. |
| **Trigger binding** | A per-repo `(intake point → pipeline)` link (`domain/repos.TriggerBinding`, table `repo_trigger_bindings`). A bound repo starts a `Pipeline` run on the matching intake event; an unbound repo is a 2xx no-op. Schedule-kind bindings additionally carry a `Schedule` (cron + notify list); at most one non-schedule binding per `(org, repo, intake point)`. |
| **Schedule** | A `Trigger binding`'s cron configuration (`domain/repos.Schedule`: `name`, UTC `cron`, `notify_user_ids`, optional `kickoff_input`). Fires per minute via `domain/pipelines.pipeline_schedule_tick`: a due binding creates a fresh yaaos-authored ticket (title = `Schedule.name`, `source="schedule"`, `source_external_id="{binding_id}:{fire_time}"` — redelivery-safe via the tickets unique constraint) and starts a run whose escalation target is `notify_user_ids`. |
| **WorkspaceAgent** | Customer-deployed Go binary in `apps/agent/` that holds source code locally and spawns workspace processes. Zero biz logic — all policy comes from control plane via AgentCommand payload. Each running instance is an agent instance tracked by a `workspace_agents` row keyed on `(org_id, instance_id)`. |
| **Pipeline** | Org-defined, user-authored sequence of stages (`PipelineDefinition`): `id`, `name`, `description`, `stages`. Discriminated on stage `kind` (`skill \| review \| action \| call`). Owned by `domain/pipelines`. |
| **Template** | A code-shipped default `PipelineDefinition` (`domain/pipelines.defaults`, six of them — `dev`, `troubleshoot`, `implementation`, `PR review`, `incremental review`, `comment response`) with a pinned uuid7 id. Never persisted or run directly — `instantiate_template` deep-copies one into a normal org `Pipeline` (fresh ids, `call` stages rewired to org copies of their callees); a new org starts with none. |
| **Stage** | One step of a `Pipeline`. Four kinds: `skill` (main-invocation, optional review loop), `review` (main invocation speaks the review contract directly), `action` (synchronous control-plane executor), `call` (expands another org pipeline at flatten time). |
| **Run** (`PipelineRun`) | One execution of a `Pipeline` against a ticket (`pipeline_runs` row). States: `queued → running ⇄ paused → completed \| failed \| killed \| cancelled`. One run in flight (`running`/`paused`) per ticket at a time — additional starts `queued`, promoted when the in-flight run reaches a terminal state. |
| **Action** | A synchronous, deterministic control-plane stage executor (`domain/actions.Action` Protocol) — no agent invocation, no parking, SAVEPOINT-wrapped. Plugins contribute `Action`s via `register_action`; `ActionStage.action_id` keys into the registry. Result persists on `stage_executions.action_result`. Shipped: `github:create_pr` (opens the PR for a yaaos-authored ticket's branch), `github:update_pr` (posts residual findings, reflects review verdicts onto the PR). |
| **Artifact** | One produced document version from a `skill` stage's completed invocation (`artifacts` row). Body immutable; `mark_final` is the only mutation. Lineage is the `(ticket_id, stage_name)` group — no separate descriptor entity. Owned by `domain/artifacts`. |
| **Residual** | A stage execution's own `Finding`s still `open` at boundary-evaluation time (`domain/findings.list_for_stage_execution`, filtered). Drives whether a `skill` stage's review loop dispatches a fix pass; feeds a stage's boundary-pause conditions (`on_blocker_residuals`/`on_should_fix_residuals`). |
| **Boundary** | The decision point after every `skill`/`review` stage's own work settles: proceed to the next stage, or pause for a human (`domain/pipelines.boundary.evaluate_boundary`). `BoundaryControl.mode` is `always_hitl` (default) \| `always_proceed` \| `conditional` (residual severities, protected-code, bucketed confidence — any one condition tripping pauses). A pause records `run_pauses.tripped`. |
| **Send-back** | A rewind of a `PipelineRun` to an earlier `SkillStage`, triggered automatically (a main skill's own `SkillReturn.outcome="send_back"`, or a residual `Finding.defect_in_artifact`) or by a human (`resolve_pause(send_back)`). Creates a NEW `stage_executions` row at the target's index carrying `RevisionContext(source="send_back")`; the run then re-runs forward. Loop-protected per run (`pipeline_runs.sendback_counts`) — a second send-back to the same target pauses instead of rewinding again (`tripped="sendback_loop"`). |
| **Escalation target** | The user(s) a pipeline pause or run-failure notification resolves to (`domain/pipelines.escalation.resolve_escalation_targets`): the kickoff actor when it's a yaaos user, else a schedule's `notify_user_ids`, else a webhook PR author's linked identity, else the org's admins — a pause or failure notification never waits on nobody. Every pause's *responders* (who may call `resolve_pause`) are this resolved set union the org's admins/owners, always. |
| **Finding verdict** | A review skill's per-finding assertion (`fixed \| still_present \| user_overrode`, or no assertion for a reply-only response) about a `Finding` it was shown as `prior_findings`. The engine applies it mechanically, for any review stage: `fixed → resolve`; `still_present → reflag` when the finding is open, `reopen` when it's resolved; `user_overrode → dismiss`. Distinct from the review-run **Verdict** below (the terminal PR-level decision). |
| **AgentCommand** | Wire message backend → WorkspaceAgent. Kinds: `ProvisionWorkspace`, `WriteFiles`, `RefreshWorkspaceAuth`, `InvokeClaudeCode`, `CleanupWorkspace`, `PushBranch`, `ConfigUpdate`, `Shutdown`, `CancelShutdown`. Defined in `apps/backend/openapi/agent-api.yaml`. Persisted in `agent_commands` (durable queue). |
| **AgentEvent** | Wire message WorkspaceAgent → backend. Kinds: `progress`, `received`, `completed_success`, `completed_failure`, `completed_skipped`. `received` cancels the lease requeue. Terminal events resume the parked run. |
| **agent_commands** | Durable command queue in Postgres. One row per dispatched AgentCommand; lifecycle: `pending → claimed → delivered → done`. Commands survive backend restarts. `attempt` increments on each lease-timeout requeue; capped at `MAX_ATTEMPT`. |
| **Command lease** | 30-second window after a command is `claimed`: the agent must POST a `received` event to flip the row to `delivered`. Without it the reaper requeues to `pending` on the next `cleanup_loop` tick. |
| **Capacity-pull** | The agent declares `new_workspaces` (capacity for new ProvisionWorkspace commands) and `workspace_ids` (idle Active workspaces) on each claim request; the backend selects a matching batch from `agent_commands`. |
| **Outbox** | DB-atomic outbound queue (`outbox_entries`). Written in the caller's transaction; drained post-commit. Backs `core/tasks.enqueue`. |
| **`core/tasks`** | Thin taskiq + Redis wrapper. The run engine's task names: `pipelines.route_run`, `pipelines.start_stage`, `pipelines.handle_agent_event`. |
| **Activity event** | High-frequency CodingAgent telemetry. Flows WebSocket → `core/sse` → SSE → UI. Never persisted; demand-pull. |
| **InstanceID** | The role-session-name extracted from the STS assumed-role ARN (`arn:aws:sts::ACCT:assumed-role/ROLE/SESSION` → `SESSION`). Derived by the backend at identity exchange; stable across pod restarts when the ECS task reuses the same session name. Stored as `workspace_agents.instance_id`. The agent learns its own `instance_id` from the exchange response — it never supplies it. |
| **VerifiedInstanceID** | Synonym for `instance_id` when emphasizing that it was derived from a backend-verified STS ARN rather than self-reported by the agent. |
| **Stale-claim guard** | `POST /api/v1/commands/{id}/events` returns `410 {"error": "stale_claim"}` when the `agent_commands` row no longer exists or is retired. Agent receives `protocol.ErrStaleClaim`, drops the event without retry, and logs `supervisor.event_stale_claim` at INFO. Backend failsafe synthesizes the in-flight failure. |
| **Failure-report-precedes-disposal** | `core/workspace.release_claim` clears `current_command_id` before the run engine is resumed. Command-to-run correlation lives on `agent_commands.run_id` — terminal events resolve their run via the command row, not the workspace row. |
| **Demand-pull** | Activity events only flow when ≥1 UI tab is subscribed. `SubscriberRegistry` issues `subscribe`/`unsubscribe` on `0 → 1` / `1 → 0` transitions. |
