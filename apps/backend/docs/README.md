# Backend docs

FastAPI service in Python 3.13. Single Docker image runs the API, serves the bundled SPA, and spawns background coroutines for review work.

## Directory shape

- `app/core/` ŌĆö infrastructure (no business logic).
- `app/domain/` ŌĆö business logic + plugin Protocols.
- `app/plugins/` ŌĆö vendor-specific implementations.
- `app/testing/` ŌĆö test-only scaffolding (excluded from prod wheel).
- `app/main.py` ŌĆö bootstrap composition order (load-bearing).
- `app/alembic/` ŌĆö hand-edited migrations using idempotent helpers.
- `bin/` ŌĆö `ci`, `sync_modules`, `check_table_access`.
- `conftest.py` ŌĆö pytest top-level fixtures.
- `pyproject.toml` ŌĆö uv + ruff config + TID251 bans.
- `tach.toml` ŌĆö generated by `bin/sync_modules`; never hand-edit.

## Running locally

`cd apps/backend && uv sync && uv run uvicorn app.main:app --reload --port 8080`. Docker runs: see [`docker/docker-compose.yml`](../../../docker/docker-compose.yml) and `docker-compose.test.yml`.

## Live API reference

FastAPI auto-generates an OpenAPI schema at `/openapi.json`. When the backend is running:

- **Swagger UI** ŌĆö `http://localhost:8080/docs`. Interactive endpoint browser; try-it-out forms.
- **Redoc** ŌĆö `http://localhost:8080/redoc`. Reference-style rendering of the same schema.

Both are derived from the live code ŌĆö always accurate, never written by hand. Hand-written docs here describe principles, architecture, and rules; the live reference is the authoritative endpoint catalogue.

## CI

`apps/backend/bin/ci` runs: `ruff format --check`, `ruff check`, `bin/sync_modules --check` (tach + layering), `bin/check_table_access`, `bin/check_doc_links` (cross-app docs), `pytest -q`.
